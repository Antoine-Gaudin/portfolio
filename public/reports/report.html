Test-Driven Bug Fixing With Auto-Verification
With 29 bug-fix sessions and 23 instances of buggy code output, Claude should write a failing test first, fix the code, then verify the test passes—all autonomously. This eliminates the painful cycles where fixes break other things (like your CSS/Tailwind losses and dev server crashes) by catching regressions before you ever see them.
Getting started: Use Claude Code with a CLAUDE.md that mandates test-first workflow, and leverage the Agent tool to spawn a parallel verification agent that runs your test suite after every edit.
Paste into Claude Code:
I have a bug: [describe bug]. Before writing any fix, first: 1) Write a test that reproduces this exact bug and fails. 2) Run the test to confirm it fails. 3) Fix the code with minimal changes—do NOT touch unrelated files. 4) Run the full test suite to confirm the fix passes AND nothing else broke. 5) Only then show me the diff. If any test fails after your fix, revert and try again. Never blame cache corruption—always investigate the actual code first.
Parallel Agents for Multi-Feature Sessions
Your most frustrating sessions involved 3-5 tasks where Claude forgot earlier requests or broke completed work while implementing the next feature. Spawning independent sub-agents per task—each working in isolation then merging—prevents the cascade of regressions that cost you hours across your Supabase persistence bugs, navbar rewrites, and scraping fixes.
Getting started: Use the Agent tool to delegate each discrete feature to a sub-agent with a scoped file list, then have a coordinator agent merge and run integration tests.

I need these changes done independently so they don't break each other. For EACH task below, spawn a separate sub-agent: Task 1: [describe]. Task 2: [describe]. Task 3: [describe]. Rules for each agent: only modify files directly relevant to your task, run the dev build after your changes, report success/failure. After all agents complete, run the full build and test suite to verify nothing conflicts. If there's a conflict, show me both approaches and let me choose.
